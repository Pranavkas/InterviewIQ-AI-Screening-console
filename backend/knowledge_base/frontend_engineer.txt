Component Architecture
Modern frontend frameworks like React encourage composing UIs from small, reusable components. Presentational (dumb) components focus on rendering, while container (smart) components manage state and logic; this separation has softened with hooks but the underlying principle of separation of concerns remains. Prop drilling through many layers is a sign that state should be lifted or moved into a context/store. Component composition (children, render props, slots) favors flexibility over deep inheritance hierarchies.

State Management
Local component state suffices for isolated UI concerns, but shared application state often needs a dedicated solution. React's Context API is good for low-frequency updates like theme or auth state, but can cause unnecessary re-renders if overused for high-frequency data. Libraries like Redux, Zustand, or Recoil provide centralized stores with predictable update patterns; Redux's unidirectional data flow (action -> reducer -> new state) makes debugging easier via time-travel tools. Server state (data fetched from APIs) is different from client state and is often better handled by libraries like React Query/TanStack Query which manage caching, revalidation, and background refetching.

Rendering Performance
Unnecessary re-renders are a common performance bottleneck. React.memo, useMemo, and useCallback prevent expensive recomputation and re-renders when dependencies haven't changed, but overusing them adds complexity without benefit. Virtualization (windowing) renders only visible items in long lists, drastically improving performance for large datasets. Code-splitting and lazy loading reduce initial bundle size, improving Time to Interactive. The browser's reflow and repaint cycle is expensive, so batching DOM updates and avoiding layout thrashing matters for smooth animations.

Browser Fundamentals
The critical rendering path describes how the browser turns HTML, CSS, and JavaScript into pixels: parsing HTML into the DOM, CSS into the CSSOM, combining them into a render tree, then layout and paint. JavaScript execution can block rendering if not deferred or made async. The event loop enables JavaScript's non-blocking I/O model: the call stack executes synchronous code, while the callback queue and microtask queue (promises) are processed between rendering frames. Understanding this explains why long synchronous tasks freeze the UI.

Accessibility
Accessible frontend development ensures interfaces are usable via keyboard navigation and screen readers. Semantic HTML elements (button, nav, header) carry built-in accessibility semantics that div-based markup lacks. ARIA attributes (aria-label, aria-live, role) supplement semantics when native elements aren't sufficient, but should not be overused ("no ARIA is better than bad ARIA"). Color contrast ratios, focus indicators, and logical tab order are essential for users with visual or motor impairments. Accessibility is not just compliance — it materially improves usability for all users.

Responsive Design
Responsive design adapts layouts to different viewport sizes using fluid grids, flexible images, and media queries. Mobile-first design starts with the smallest viewport and progressively enhances for larger screens, generally resulting in leaner CSS. CSS Grid excels at two-dimensional layouts while Flexbox is suited to one-dimensional alignment. Relative units (rem, %, vw/vh) scale better across devices than fixed pixel values. Container queries allow components to respond to their container's size rather than only the viewport.

Build Tooling
Modern build tools like Vite and webpack bundle and transform source code for the browser: transpiling newer JavaScript/TypeScript syntax (via Babel/SWC), bundling modules, tree-shaking unused code, and minifying output. Vite leverages native ES modules during development for near-instant hot module replacement, while bundling more traditionally for production. Source maps let developers debug minified production code against original sources. Environment-based configuration lets the same codebase build differently for development, staging, and production.
